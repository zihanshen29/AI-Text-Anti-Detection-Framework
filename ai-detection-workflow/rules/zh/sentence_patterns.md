# 中文句法模式规则库 (Chinese Sentence Patterns)

> **由 Layer 0 (Discovery) 在 Step 4 扫描时加载。**
> **补充：** 与 `ai_cliches.md`（词汇层信号）互补。本文件处理**句法和结构层**信号。
> **Layer 2 (Execution) 不读此文件。**

## 与 ai_cliches.md 的关系

`ai_cliches.md` 捕获**词汇层**信号——具体的 AI 套话、黑话、程度副词。本文件捕获**句法和结构层**信号——句长分布、欧化句式、长定语堆叠、LLM 特有的节奏习惯。

中文检测器的技术现实是：词汇层比句法层更强（见 `ai_cliches.md` 开头说明）。但句法层不是不重要，而是**在词汇信号被清除之后，句法信号就成为主要区分点**。一篇已经替换掉所有"赋能/抓手"的论文如果仍然句长均匀、定语过长，检测器仍能判定为 AI。

因此这份规则库的使用顺序通常是：Layer 1 的 Round 1-3 主处理词汇层（命中集中且修改成本低），Round 3-4 处理本文件的句法层（修改成本较高但对词汇清除后的残余信号起决定作用）。

## 阅读格式

每条规则包含：

- **S-NN** —— 规则编号（S for Syntax）。
- **检测方式** —— 是文本统计（需要计算）还是模式匹配（可 grep）。
- **为什么被标记** —— 一句话解释。
- **体裁相关性** —— 不同体裁下的适用度。
- **修改方向** —— 一般指导。

## 一、句长分布与节奏

### S-01: 句长过度均匀

- **检测方式：** 按中文标点（。！？）切分句子，计算每 500 字窗口内句子的字数分布。若标准差 < 8 且均值在 18–30 之间，即 hit。
- **为什么被标记：** 中文 LLM 输出有强烈的"中庸句长"偏好，大量产出 20–25 字句。真人写作的句长分布自然地包含很短句（5-10 字）和很长句（40+ 字）的混合。
- **强敏感检测器：** 知网 AIGC（这是它的核心统计特征之一），笔灵。
- **体裁相关性：** 学术论文（★★★），报告（★★★），商业文案（★★），新闻（★），文学（★）。
- **修改方向：** 主动引入句长变化。把一个 20 字长句拆成 5 字 + 30 字两句；把两个 15 字中句合并成一个 40 字复句。不是随机变化，是**有意识地打破均匀分布**。注意不要矫枉过正变成 S-02 的问题。

### S-02: 短句群堆叠（对应英文 P-09）

- **检测方式：** 扫描任意三个连续句子，若每句都 < 12 字，即 hit。
- **为什么被标记：** 这是"AI 降重工具"产物的典型特征——把长句机械劈成多个短句，看起来有节奏但学术体裁里不会这样写。真人论文的短句通常单独出现作为强调，不会连续成串。
- **强敏感检测器：** 笔灵（专门针对降重工具输出训练），知网 AIGC。
- **体裁相关性：** 学术论文（★★★ 严重），技术文档（★★★），新闻（★，可接受），文学（○，可保留）。
- **修改方向：** 把连续短句中至少一对合并成复句，保留一个短句作为强调。"注意力先变。翻滚到 -5 度。俯仰峰值 +26 度。" → "注意力变化先于基体高度，翻滚在接近 818 步时达到 -5 度，俯仰则在此之前达到 +26 度的峰值。"

### S-03: 长句过度嵌套

- **检测方式：** 扫描 > 60 字的句子，统计其中逗号分隔的从句/短语数量。若单句含 > 5 个逗号分隔单元，即 hit。
- **为什么被标记：** LLM 中文输出有把多个意思塞进一个长句的倾向，用逗号串联取代句号。真人写作会在适当位置断句。
- **强敏感检测器：** 知网 AIGC、万方（长句 POS 特征）。
- **体裁相关性：** 学术论文（★★★），法律文书（★，合理），技术文档（★★），文学（★★★，但风格化用法可能合理）。
- **修改方向：** 在主要语义切换点断句。长句通常能在"，同时"、"，从而"、"，这使得"、"，以便"这些连接处自然拆开。

## 二、欧化句式

欧化句式是 LLM 中文输出的独特标签。因为训练语料里包含大量翻译文本和英文风格迁移样本，LLM 倾向产生"一眼能看出是从英语思维硬转过来"的中文句法。真人中文写作里这些模式低频得多。

### S-04: 长定语堆叠

- **检测方式：** 搜索形如 `[A 的 B 的 C 的 ... X]` 的名词短语，若单个名词短语包含 ≥ 3 个"的"字定语修饰，即 hit。例："基于深度学习的多模态的实时的目标检测算法"。
- **为什么被标记：** 中文真人写作不会堆叠这么多"的"。这是英语长复合名词短语（`the deep-learning-based multimodal real-time object detection algorithm`）的直译产物，LLM 生成时常见。
- **强敏感检测器：** 知网 AIGC、万方、维普。
- **体裁相关性：** 学术论文（★★★），技术文档（★★★），新闻（★★），文学（★）。
- **修改方向：** 拆成多个动词短语或用逗号分层。"基于深度学习的多模态的实时的目标检测算法" → "一种基于深度学习的实时目标检测算法，支持多模态输入"。

### S-05: "被"字句过量

- **检测方式：** 统计"被"字句密度。若每 500 字 > 4 个"被"字句即 hit；连续两句都是被动语态是强信号。
- **为什么被标记：** 真人中文倾向主动语态或无主语句。LLM 受英文被动语态影响，产出大量"X 被 Y 处理"、"Y 被 Z 提出"的句子。
- **强敏感检测器：** 知网 AIGC、笔灵。
- **体裁相关性：** 学术论文（★★★ 严重），新闻（★★），法律（★，合理），文学（★）。
- **修改方向：** 改为主动语态或无主语结构。"该方法被 Smith 提出" → "Smith 提出了该方法"；"结果被分为三类" → "将结果分为三类"或"结果分为三类"。

### S-06: "对...进行..."套句

- **检测方式：** 搜索 `对...进行`、`对...作出`、`对...加以`、`对...开展` 模式。若每 500 字 > 3 次即 hit。
- **为什么被标记：** LLM 用这种迂回结构填充动作动词。真人中文直接用动词。
- **强敏感检测器：** 知网 AIGC、万方。
- **体裁相关性：** 学术论文（★★★），政府文件（★，合理），技术文档（★★），文学（★）。
- **修改方向：** 直接用动词替换。"对数据进行分析" → "分析数据"；"对结果进行验证" → "验证结果"；"对方法加以改进" → "改进方法"。

### S-07: "作为...的..."插入语

- **检测方式：** 搜索句首或句中的 `作为 X 的 Y`、`作为 X，...` 插入模式。密度 > 2 per 1000 字即 hit。
- **为什么被标记：** 英文 `As a X, Y does Z` 结构的直译。中文真人写作倾向分句表达："作为一种重要的方法，它..." → "它是一种重要的方法，..."。
- **强敏感检测器：** 笔灵、PaperPass。
- **体裁相关性：** 学术论文（★★★），报告（★★），新闻（★），文学（★）。
- **修改方向：** 改为独立判断句或拆分成两句。

### S-08: "它"字指代滥用

- **检测方式：** 统计"它"、"它们"的密度。若每 500 字 > 5 次（人称代词作主语）即 hit。
- **为什么被标记：** 英文 `it` 和 `they` 的直译。中文真人写作倾向**省略主语**或**复现名词**，而不是用"它"连续指代。连续三句都用"它"做主语是极强信号。
- **强敏感检测器：** 知网 AIGC、笔灵。
- **体裁相关性：** 学术论文（★★★），技术文档（★★★），文学（★）。
- **修改方向：** 省略主语让连接从上下文推导，或重复具体名词。"该算法具有 X 优势。它也有 Y 特点。它在 Z 场景下表现良好。" → "该算法具有 X 优势，也有 Y 特点，在 Z 场景下表现良好。"

### S-09: "使得...成为..."套句

- **检测方式：** 搜索 `使得 ... 成为 ...`、`使 ... 变得 ...`、`这使得 ...`。密度 > 2 per 1000 字即 hit。
- **为什么被标记：** 英文 `makes X Y` 的直译（`This makes X possible`）。中文更自然的说法是直接陈述结果。
- **强敏感检测器：** 知网 AIGC、万方。
- **体裁相关性：** 学术论文（★★★），报告（★★），新闻（★）。
- **修改方向：** 改为直接陈述。"这使得实时处理成为可能" → "这支持实时处理"或"因此可以实时处理"。

### S-10: 三段式从句翻译腔

- **检测方式：** 搜索 `在 ... 方面，... 表明 ...` 这类形式化三段结构；或以 `从 ... 的角度来看` 开头的长句。
- **为什么被标记：** 英文 `In terms of X, the data show Y` 结构的直译。中文真人写作更简洁。
- **强敏感检测器：** 笔灵、知网 AIGC。
- **体裁相关性：** 学术论文（★★），报告（★★），新闻（★）。
- **修改方向：** 去掉框架词，直接陈述。"从实验数据的角度来看，结果表明..." → "实验数据显示..."。

## 三、段落结构与过渡

### S-11: 完美"三段论"段落

- **检测方式：** 扫描段落长度超过 200 字的段落，若开头句是总述（通常 15-25 字）、中间句为并列展开（通常 2-4 个结构相似的子句）、结尾句是小结（通常 20-30 字），即 hit。这是模式识别，需要语义判断辅助。
- **为什么被标记：** LLM 的段落构造模板。真人段落结构更不规则——可能没有总述，或没有小结，或从具体例子开始。
- **强敏感检测器：** 知网 AIGC（段落结构特征）、笔灵。
- **体裁相关性：** 学术论文（★★★），政府公文（★★ 合理），报告（★★），文学（★）。
- **修改方向：** 打破模板。删掉总述句让段落直接进入论证；或把小结句合并到下一段的开头；或把中间的并列展开改成递进推理。

### S-12: 段落开头统一模式

- **检测方式：** 扫描相邻 3-4 段的开头句式。若每段都以相同类型开头（如每段都以"首先/其次/再者"、"一方面/另一方面"、"X 方面"），即 hit。
- **为什么被标记：** LLM 产出多段时倾向使用模板化段落开头。真人写作段落开头更多样。
- **强敏感检测器：** 知网 AIGC、万方。
- **体裁相关性：** 学术论文（★★★），技术文档（★★★），政府公文（★ 合理），文学（○）。
- **修改方向：** 改变一半段落的开头句式。把"首先...其次...再次..."式模板拆成内容主导的自然过渡。

### S-13: 过度显式的段内逻辑标记

- **检测方式：** 在单个段落内搜索 `然而`、`但是`、`因此`、`所以`、`此外`、`同时` 等转折/因果词。若单段超过 120 字且含 ≥ 3 个这类逻辑词，即 hit。
- **为什么被标记：** LLM 倾向于用显式连接词标记每个逻辑转折，而真人中文写作常常依靠语序和隐含推导。
- **强敏感检测器：** 笔灵、PaperPass。
- **体裁相关性：** 学术论文（★★），议论文（★★），法律（★ 合理），文学（★）。
- **修改方向：** 删除冗余的逻辑标记词，保留必要的。一般一段只需保留 1-2 个关键转折词。

## 四、LLM 特有的中文节奏

### S-14: 对称句的规律出现

- **检测方式：** 扫描文档中结构对称的并列句（如"X 是 A, Y 是 B" 或"不但 X, 而且 Y"）。若密度 > 3 per 1000 字，即 hit。
- **为什么被标记：** 参见 `ai_cliches.md` C-09（不仅...而且 套句）——这里侧重的是更广义的对称句式，包括无标记的对称（"X 提升了 A, Y 改善了 B"）。
- **强敏感检测器：** 知网 AIGC、笔灵。
- **体裁相关性：** 学术论文（★★），政府公文（★★），议论文（★★），文学（★ 视风格）。
- **修改方向：** 打破部分对称结构。让一半的对称句变成非对称表述。

### S-15: 全文段落长度均匀

- **检测方式：** 统计全文段落字数分布。若所有段落字数标准差 / 均值 < 0.3（变异系数低），即 hit。
- **为什么被标记：** LLM 倾向产出长度均匀的段落（通常 150-250 字）。真人写作会有很短的段落（50 字以下，用于强调或过渡）和较长的段落（400+ 字，用于密集论证）混杂。
- **强敏感检测器：** 知网 AIGC（少数会用段落级特征）。
- **体裁相关性：** 学术论文（★★），报告（★★），新闻（★），文学（★）。
- **修改方向：** 在关键论点处创造一到两个极短段落（单句段），用于强调或过渡。让全文段落长度分布更不均匀。

### S-16: 句末"的"字收尾

- **检测方式：** 统计以"...的。" 结尾的句子比例。若占所有句子 > 15%，即 hit。
- **为什么被标记：** LLM 中文输出有"是...的"句式偏好，句末"的"字收尾频率高。真人写作句式收尾更多样。
- **强敏感检测器：** 笔灵、PaperPass。
- **体裁相关性：** 学术论文（★★），新闻（★★），文学（★）。
- **修改方向：** 改写部分"是 X 的"为直接判断。"该结果是正确的" → "该结果正确"或"这一结果成立"。

### S-17: "等等"、"诸如此类"、"之类"的冗余收尾

- **检测方式：** 搜索 `等等`、`诸如此类`、`之类`、`...等` 作为句末收尾。若单文档 > 3 次即 hit。
- **为什么被标记：** LLM 在列举后补充"等等"以示"列举不完"。真人写作要么列完，要么明确说"还有其他类型"，很少用"等等"。
- **强敏感检测器：** 笔灵、知网 AIGC。
- **体裁相关性：** 学术论文（★★★），技术文档（★★），新闻（★），文学（★ 视口语化）。
- **修改方向：** 删除冗余的"等等"，或明确补全。"包括 A、B、C 等等" → "包括 A、B、C" 或 "包括 A、B、C 以及其他六种情况"。

## 五、摘要/引言/结论的结构信号

### S-18: 摘要"三段论"结构

- **检测方式：** 检查摘要是否严格遵循"背景/问题 → 方法 → 结果"三段且每段字数近似相等。
- **为什么被标记：** LLM 生成摘要有模板化倾向。真人摘要也有三段论但比例更不均匀（方法通常最长，背景最短，或有经验的作者会打破模板以突出创新点）。
- **强敏感检测器：** 知网 AIGC、PaperPass（专门训练摘要识别）。
- **体裁相关性：** 学术论文摘要（★★★），其他体裁不适用。
- **修改方向：** 调整三段比例让方法部分显著长于其他两段；或合并背景与方法，把结果单独成段；或从一个具体研究问题切入代替背景铺垫。

### S-19: 引言的背景铺垫冗长

- **检测方式：** 检查引言第一段。若第一段超过 150 字且不含任何具体研究问题或文献引用，即 hit。
- **为什么被标记：** LLM 引言常以大背景铺垫开头（见 `ai_cliches.md` C-02）。真人论文引言常在第一段就进入具体问题或引用关键文献。
- **强敏感检测器：** 知网 AIGC、万方。
- **体裁相关性：** 学术论文（★★★）。
- **修改方向：** 删除或大幅压缩背景铺垫，直接进入具体研究问题。引言第一段理想长度 100-150 字，包含至少一个具体问题或关键引用。

### S-20: 结论的"三段式"收尾

- **检测方式：** 检查结论章节是否严格按"总结贡献 / 承认局限 / 展望未来"三段论组织，且每段形式化程度高（用词可预测）。
- **为什么被标记：** LLM 结论模板化严重。真人结论有的只有两段（没有专门"展望"），有的把三部分融合在一段，有的加入意外的反思内容。
- **强敏感检测器：** 知网 AIGC、笔灵。
- **体裁相关性：** 学术论文（★★★）。
- **修改方向：** 打破三段模板。加入具体的"如果能重新做一次会怎么改"这种反思性内容（很多评分表明确奖励这一点）；或把局限性融入贡献讨论而非单独成段。

## 与 ai_cliches.md 的联合使用

Layer 0 扫描时，**同时**加载 `ai_cliches.md`（C-NN 规则）和本文件（S-NN 规则）。扫描到一个段落同时触发 C-NN 和 S-NN 时（例如一个段落既含"赋能/抓手"又句长均匀），在 discovery.md 里合并记录，避免重复计数。

Layer 1 规划轮次时，先处理词汇层 C 规则（单点替换，风险低，修改快），再处理本文件的句法层 S 规则（需要重构句子和段落，风险较高）。

通用优先级：若不知道目标检测器，先处理 **S-01（句长均匀）、S-04（长定语）、S-05（被字句）、S-06（对...进行）、S-08（它字指代）** 这五条——所有主流中文检测器都吃这些信号，且修改技术相对成熟。

## 新句法模式占位槽

如 Layer 0 扫到不在 S-01 ~ S-20 的 AI 味句法模式，记录在 discovery.md 的 Novel patterns 部分。LLM 中文生成的句法习惯随训练语料更新而变化；规则库应持续迭代。
